## DESCRIPTION OF TASK

In this task, I implemented cloud security features using Google Cloud Platform (GCP). The main objective of this project was to configure secure cloud storage, implement IAM (Identity and Access Management) policies, and enable data encryption to protect cloud resources and stored files.

I started by creating and configuring a Cloud Storage bucket in Google Cloud Storage. The bucket acts as a secure storage location for files and documents uploaded to the cloud environment.

To improve storage security, I configured Uniform Bucket-Level Access, which ensures centralized permission management for all objects stored inside the bucket. This prevents unauthorized users from changing object-level permissions individually.

I also verified that encryption was enabled using Google-managed encryption keys. This feature automatically encrypts all stored data at rest, providing protection against unauthorized access and improving overall cloud security.

Next, I implemented IAM policies using Google Cloud IAM. IAM helps control access to cloud resources by assigning roles and permissions to users. In this project, the Owner role was assigned for administrative control and secure resource management.

Additionally, sample files such as PDF documents and resume files were uploaded into the storage bucket to demonstrate secure cloud storage functionality.

This task provided practical knowledge about cloud security implementation, secure storage configuration, IAM role management, and data encryption techniques used in real-world cloud environments.

Cloud security is an important aspect of cloud computing because it helps organizations protect sensitive information, control user access, and ensure secure data management.
